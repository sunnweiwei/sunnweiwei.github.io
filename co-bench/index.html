CO-Bench: Benchmarking Language Model Agents in Algorithm Search for Combinatorial Optimization

Benchmark Algorithm Search
Weiwei Sun, Shengyu Feng, Shanda Li, Yiming Yang
Carnegie Mellon University
Paper Code Home

Abstract

Although LLM-based agents have attracted significant attention in domains such as software engineering and machine learning research, their role in advancing combinatorial optimization (CO) remains relatively underexplored. This gap underscores the need for a deeper understanding of their potential in tackling structured, constraint-intensive problems—a pursuit currently limited by the absence of comprehensive benchmarks for systematic investigation. To address this, we introduce CO-Bench, a benchmark suite featuring 36 real-world CO problems drawn from a broad range of domains and complexity levels. CO-Bench includes structured problem formulations and curated data to support rigorous investigation of LLM agents. We evaluate multiple agentic frameworks against established human-designed algorithms, revealing the strengths and limitations of existing LLM agents and identifying promising directions for future research.

About CO-Bench

What is CO-Bench?

CO-Bench is a comprehensive benchmark designed to evaluate Large Language Model (LLM) agents in developing efficient algorithms for combinatorial optimization problems. Unlike existing benchmarks that focus on solution correctness for small instances, CO-Bench emphasizes end-to-end algorithm design from abstract problem descriptions, evaluating agents' ability to discover efficient and potentially novel algorithms.

Problem Categories

CO-Bench includes 36 real-world problems spanning 8 diverse categories:

Routing

Vehicle Routing

Problems involving optimal route planning for vehicles, including traveling salesman problem (TSP) and vehicle routing with constraints.

Scheduling

Job Scheduling

Task assignment and timing problems, such as job shop scheduling and resource allocation under temporal constraints.

Packing

Bin Packing

Space optimization problems including 1D/2D/3D bin packing and knapsack variants with various constraints.

Graph Theory

Graph Problems

Classic graph optimization tasks such as max cut, vertex cover, and graph coloring problems.

Production

Production Planning

Manufacturing and production optimization including facility location and production scheduling.

Logistics

Supply Chain

Warehouse management, inventory optimization, and distribution network design problems.

Network Design

Network Optimization

Communication network design, flow optimization, and infrastructure planning problems.

Bioinformatics

Sequence Alignment

Biological sequence analysis and phylogenetic tree construction problems.

Key Features

  • Diverse Problem Set: 36 real-world problems from 8 different domains with varying complexity levels
  • Structured Formulations: Each problem includes abstract descriptions, constraints, and curated datasets
  • End-to-End Evaluation: Focus on autonomous algorithm design rather than just solution correctness
  • Rigorous Benchmarking: Comparison with classical solvers and best-known solutions from literature
  • Scalable Testing: Multiple instance sizes to evaluate algorithm efficiency and scalability

Example Problems

Here are some representative problems from CO-Bench:

Traveling Salesman Problem (TSP)

Category: Routing
Description: Find the shortest route visiting all cities exactly once and returning to the origin.
Complexity: NP-hard
Instance Sizes: 50-1000 cities

Capacitated Vehicle Routing (CVRP)

Category: Routing
Description: Optimize delivery routes for vehicles with capacity constraints.
Complexity: NP-hard
Instance Sizes: 20-200 customers

Job Shop Scheduling

Category: Scheduling
Description: Schedule jobs on machines to minimize makespan with precedence constraints.
Complexity: NP-hard
Instance Sizes: 10x10 to 50x20 (jobs×machines)

3D Bin Packing

Category: Packing
Description: Pack 3D items into minimum number of bins respecting dimension constraints.
Complexity: NP-hard
Instance Sizes: 50-500 items
